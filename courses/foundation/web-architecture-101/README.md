CRUD Operation flow
1- CREATE - user submits a recipe -> Front end sends POST ->backend inserts DB ->
frontend shows result
//

2- READ - user views recipe -> Front end sends GET -> backend queries DB ->
frontend displays result
//

3- UPDATE - user edits recipe -> frontend sends PUT -> backend updates DB ->
frontend shows updated result
//

4- DELETE - user deletes recipe -> frontend sends DELETE ->backend removes from DB ->
frontend updates list and displays to user.
